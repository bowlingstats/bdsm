Bowling Database/Statistics Management

The guts of the project is most definately the database, so I'll start by showing you it's basic shape. Next I'll move on to layout the scripts that will make up the backbone of the project and what functionality they will provide.

Database Layout:
Table Name Column Type Description
Users uid int(4) unique ID of a user, auto increments
username char(8) username of a bowler on this system
password char(64) password of a bowler
games game_id int(4) id of games played. used to keep games straight in other tables
player_id int(4) same value as users.uid, used to track individual bowler scores in a game
scores game_id int(4) same value as games.game_id
player_id int(4) same value as games.player_id
frame int(2) which frame is this for player_id in game_id
b1 int(2) Default not null. Score of ball 1 in this frame
b2 int(2) Default null. Score of ball 2 in this frame
b3 int(2) Default null. Score of ball 3 in this frame
pinfall game_id int(4) see scores.game_id
player_id int(4) see scores.player_id
frame int(2) see scores.frame
b1 char(20) Is a comma seperated list of the pins still standing after the first ball.
b2 char(20) Is a comma seperated list of the pins still standing after the second ball.
b3 char(32) Is a comma seperated list of the pins still standing after the third ball.

Scripts:

game: Takes a value for how many players are in this game. Creates a HTML table row with the scoring form inside it. The form submits to gamesubmit. The form is verified and scored by JavaScript.

gamesubmit: Inserts, updates, or deletes infromation into the database.

gameedit: Gathers game info from the database by game_id, fills the values for an HTML form. Submits to gamesubmit. It also offers the option of deleting the game being edited.

result: Takes a value game_id and displays that game in HTML tables.

statistics: Will probably be written in PERL, for it's heavy lifting. Querys the database, finding statistics for strikes, spares, spare conversions, etc.

standings: Ranks users by averages, total pinfall, maybe other criteria.
