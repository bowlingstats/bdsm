Bowling Database/Statistics Management

The guts of the project is most definately the database, so I'll start by showing you it's basic shape. Next I'll move on to layout the scripts that will make up the backbone of the project and what functionality they will provide.

Database Layout:
Table Name Column Type Description
Users uid smallint(4) unique ID of a user, auto increments
username varchar(8) username of a bowler on this system
password varchar(64) password of a bowler
name varvarchar(32) a handle for the bowler
admin smallint(1) governs what class of user this is
games game_id smallint(4) id of games played. used to keep games straight in other tables
player_id small(4) same value as users.uid, used to track individual bowler scores in a game
score smallint(3) score of the game, for quick reference
date datetime keeps the date of the game
location varchar(50) The location the game was played at(eg Suburban Lanes)
scores game_id small(4) same value as games.game_id
player_id smallint(4) same value as games.player_id
frame smallint(2) which frame is this for player_id in game_id
b1 smallint(2) Default not null. Score of ball 1 in this frame
b2 smallint(2) Default null. Score of ball 2 in this frame
b3 smallint(2) Default null. Score of ball 3 in this frame
pinfall game_id smallint(4) see scores.game_id
player_id smallint(4) see scores.player_id
frame smallint(2) see scores.frame
b1 varchar(20) Is a comma seperated list of the pins still standing after the first ball.
b2 varchar(20) Is a comma seperated list of the pins still standing after the second ball.
b3 varchar(32) Is a comma seperated list of the pins still standing after the third ball.

Scripts:

index: The splash page, lists all users, lists 10 best averages, and 10 best games<

header/footer: Connects to the database, and includes functions useed on each page: player(), user(), admin() creates links at top of pages.

login/logout: Used to create and destory user sessions.

scoring.js: has javascript functions for scoring games as they are entered/edited.

game: Takes a value for how many players are in this game. Creates a HTML table row with the scoring form inside it. The form submits to gamesubmit. The form is verified and scored by JavaScript.

gamesubmit: Inserts, updates, or deletes infromation into the database.

gameedit: Gathers game info from the database by game_id, fills the values for an HTML form. Submits to gamesubmit. It also offers the option of deleting the game being edited.

result: Takes a value game_id and displays that game in HTML tables.

statistics: Will probably be written in PERL, for it's heavy lifting. Querys the database, finding statistics for strikes, spares, spare conversions, etc.

usersgames: Takes a value uid and lists games for the player with that uid in the users table. Allows some sorting, and displays rudimentary statistics.

users: Adds, edits, or deletes users. When a user is deleted all their games go too.

standings: Ranks users by averages, total pinfall, maybe other criteria.
