A brief intro to Subversion(SVN)

Subversion is a tool that simplifies collaberation on a software project. You use a SVN client to "check out" files from a SVN server. You edit the local files, and when you're done doing that you will use your SVN client to "commit" the changes to the server. The server logs all the changes and at any point you can check out code from previous revisions.

To familiarize you with how to use SVN on this project I'm going to give you a short list of things to do.

Check out a copy of the SVN repository to work on:

  1. Download TortoiseSVN, and install it. It will ask you to restart, you'll need to. Now when you right-click you'll be able to access SVN options for folders and files.
  2. Create a folder where you'd like to check the bdsm project files into. Note: It is important that this folder be empty, or you will wind up adding unwanted files into the SVN repository.
  3. Right-click on the folder you have created. Choose SVN Checkout...
  4. There are several fields here for the URL of repository enter https://svn.sourceforge.net/svnroot/bdsm/ and for Checkout Directory there are two checkboxes, leave them blank. For the Revision options we want to select HEAD. Click Ok
  5. This may take a moment, it will mention what files it grabbed. Go ahead and close this window once it's done.(click ok)
Modify and commit files back to the SVN repository:
  1. Open the folder you made you'll see two folders roadmap and test. Open the test folder.
  2. Make a new text file yourname.txt where "yourname" is your name.
  3. Write something in yourname.txt, and save those changes.
  4. Right click on yourname.txt, under TortoiseSVN choose add... This adds the file to files that should be uploaded to the online SVN repository.
  5. Right click on yourname.txt, and click on SVN Commit.
  6. In the window that comes up it gives you a short review of files to be "commited," that is uploaded.
  7. You will be prompted for your SourceForge username and password.(The very first time you do this you'll be asked to accept a certificate. Please do so.)
  8. It will show you a short dialog about the status of the files you have commited. Click ok.
If you were to delete the contents of the folder you created you would be able to check the files out again and all would be as you left it.

It is vital that before you start editing that you check out a fresh copy of the SVN repository, this way you will not obliterate updates another developer may have made when you commit your work. This is easily accomplished by right clicking on the folder you created, and choosing SVN Update. There is much more robust documentation out there than this, but this should help get your feet wet.
